# 超简单使用说明

## 1. 改配置

打开 `config.yaml`。

填写图片文件夹地址, 大模型配置, 然后

```yaml
mode: global_tags_only

run:
  parallel: 4
```

`mode` 有两个值：

- `global_tags_only`：只做全局标签，不拉框，速度更快。
- `boxes_and_tags`：拉框 + 全局标签，每张图片只调用一次大模型。

`parallel` 是并行数量：

- 正常用 `4`
- 如果出现 `429 Too Many Requests`，改成 `2` 或 `1`
- 如果运行超时，先保持 `mode: global_tags_only`，并把图片尺寸用 `960x540`

## 2. 递归目录读取

脚本会自动读取目录里面的图片，也会读取子文件夹里的图片。

## 3. 运行

在 PowerShell 进入项目目录：

```powershell
python .\preannotate.py
```

## 4. 中断后继续

直接再运行一次：

```powershell
python .\preannotate.py
```

已经成功的图片会自动跳过。

## 5. 输出文件

结果在：

```text
.\output
```

文件名类似：

```text
labelu_163_Qwen_Qwen3.5-4B.json
```

这个 JSON 可以导入 LabelU 做预标注。

如果同名 JSON 已经存在，脚本不会覆盖，会自动生成 `_1`、`_2` 这样的新文件名。

正在跑的输出文件名会记录在 `output/checkpoint.jsonl` 里，下次续跑会继续写同一个 JSON。

## 6. 重新全部跑


如果想清空进度重新跑，

第一种方法删除`checkpoint.jsonl`和`failed.jsonl`

第二种方法把 `config.yaml` 改成：

```yaml
run:
  overwrite_checkpoint: true
```

跑完后建议再改回：

```yaml
run:
  overwrite_checkpoint: false
```
